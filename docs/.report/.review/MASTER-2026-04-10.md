### 根因分析

`rusty-claude-cli/src/main.rs` 与 `rust/crates/commands/src/lib.rs` 是**两个最大且持续高频率修改**的文件，近期插入了大量新功能（内部进度报告、reasoning effort、新 slash 命令等），导致几乎所有基于旧版本的硬编码行号**集体向下漂移 200–500 行**。这是本轮审校中 CRITICAL 问题高度集中的首要原因。

**建议**：对这两个文件中的锚点，统一改为：
- 使用 `grep -n 'fn <symbol>'` 得到的当前动态行号；或
- 改用**不带行号的文件级链接** + 函数名文字说明，降低维护成本。

## 跨报告一致性观察

- **无直接概念矛盾**：不同报告对同一机制（如 `PermissionMode` 五级枚举、`ConversationRuntime::run_turn`、自动压缩阈值等）的描述基本一致。
- **MCP 概念边界需澄清**：`07-what-are-tools.md` 描述的是 Rust 中的 `McpToolRegistry` / `McpServerManager`（tools/resources），而 `46-mcp-skills.md` 描述的是上游 TypeScript 中的 `skill://` URI → `Command` 转换，两者名称相近但内涵不同，建议在 `07-what-are-tools.md` 中加一句注脚区分。
- **Sleep 工具 vs Proactive 模式**：`07-what-are-tools.md` 中的 `Sleep` 工具在 Rust 中确实存在，但它只是 `tokio::time::sleep` 包装，**没有**与 Proactive 自主代理模式绑定；`44-proactive.md` 已声明该模式在 Rust 中缺失，两者并不矛盾。
